3//! This module provides VRF operations using AWS CloudHSM, Amazon's managed hardware
4//! security module service. AWS CloudHSM offers FIPS 140-2 Level 3 validated HSMs
5//! running in AWS infrastructure with single-tenant access.

7//! # Current Status

15//! - PKCS#11 library: `/opt/cloudhsm/lib/libcloudhsm_pkcs11.so`
16//! - Network connectivity to cluster ENIs in VPC

76//! # Setup Instructions
77//!
78//! ## 1. Provision CloudHSM Cluster
79//!
80//! ```bash
81//! # Create cluster
82//! aws cloudhsmv2 create-cluster \
83//!     --hsm-type hsm1.medium \
84//!     --subnet-ids subnet-abc123 subnet-def456 subnet-ghi789
85//!
86//! # Initialize cluster (one-time)
87//! aws cloudhsmv2 create-hsm \
88//!     --cluster-id cluster-abc123 \
89//!     --availability-zone us-east-1a
90//!
91//! # Wait for HSM to be active
92//! aws cloudhsmv2 describe-clusters --filters clusterIds=cluster-abc123
93//! ```
94//!
95//! ## 2. Install CloudHSM Client
96//!
97//! ```bash
98//! # Amazon Linux 2 / RHEL / CentOS
99//! wget https://s3.amazonaws.com/cloudhsmv2-software/CloudHsmClient/EL7/cloudhsm-client-latest.el7.x86_64.rpm
100//! sudo yum install -y ./cloudhsm-client-latest.el7.x86_64.rpm
101//! sudo yum install -y cloudhsm-client-pkcs11
102//!
103//! # Ubuntu / Debian
104//! wget https://s3.amazonaws.com/cloudhsmv2-software/CloudHsmClient/Bionic/cloudhsm-client_latest_amd64.deb
105//! sudo dpkg -i cloudhsm-client_latest_amd64.deb
106//! sudo apt install -y cloudhsm-client-pkcs11
107//! ```
108//!
109//! ## 3. Configure Client
110//!
111//! ```bash
112//! # Configure cluster connection
113//! sudo /opt/cloudhsm/bin/configure -a <cluster-ENI-IP>
114//!
115//! # Start client daemon
116//! sudo systemctl start cloudhsm-client
117//! sudo systemctl enable cloudhsm-client
118//!
119//! # Verify connectivity
120//! /opt/cloudhsm/bin/cloudhsm_mgmt_util /opt/cloudhsm/etc/cloudhsm_mgmt_util.cfg
121//! ```
122//!
123//! ## 4. Create Crypto User (CU)
124//!
125//! ```bash
126//! # Login as CO (Crypto Officer)
127//! aws cloudhsmv2 describe-clusters | grep CustomerCaCertificate
128//! /opt/cloudhsm/bin/cloudhsm_mgmt_util
129//!
130//! # In cloudhsm_mgmt_util:
131//! loginHSM CO admin <password>
132//! createUser CU vrf_user <password>
133//! quit
134//! ```

184//! # Security Best Practices
185//!
186//! ## Credential Management
187//! - ✅ Store credentials in **AWS Secrets Manager**
188//! - ✅ Rotate passwords regularly (90 days recommended)
189//! - ✅ Use **IAM roles** for EC2/ECS/Lambda permissions
190//! - ❌ Never hardcode credentials in source code
191//! - ❌ Never commit credentials to version control
192//!
193//! ## Network Security
194//! - ✅ Deploy HSMs in **private subnets**
195//! - ✅ Use **Security Groups** to restrict access
196//! - ✅ Enable **VPC Flow Logs** for audit
197//! - ✅ Use **PrivateLink** for AWS service access
198//! - ❌ Never expose HSM ENIs to public internet
199//!
200//! ## Key Management
201//! - ✅ Set keys as **non-extractable** (CKA_EXTRACTABLE=false)
202//! - ✅ Set keys as **sensitive** (CKA_SENSITIVE=true)
203//! - ✅ Enable **CloudWatch** logging for key operations
204//! - ✅ Backup keys using **CloudHSM backup** feature
205//! - ✅ Test **disaster recovery** procedures
206//!
207//! ## Monitoring
208//! - ✅ Monitor CloudWatch metrics: `NumberOfUsers`, `HSMTemperature`
209//! - ✅ Set alarms for `HSMUnhealthy` state
210//! - ✅ Track `ClientConnections` and session counts
211//! - ✅ Log all crypto operations for compliance

226//! # Error Handling
227//!
228//! Common CloudHSM errors and recovery:
229//!
230//! ## CKR_USER_NOT_LOGGED_IN
231//! - **Cause**: Session timeout or invalid credentials
232//! - **Recovery**: Re-authenticate with CU credentials
233//!
234//! ## CKR_DEVICE_ERROR
235//! - **Cause**: HSM hardware issue or network failure
236//! - **Recovery**: Client auto-fails over to another HSM in cluster
237//!
238//! ## CKR_SESSION_HANDLE_INVALID
239//! - **Cause**: Session expired or client restarted
240//! - **Recovery**: Create new session
241//!
242//! ## Connection Timeout
243//! - **Cause**: Network issue or client daemon down
244//! - **Recovery**: Check `cloudhsm-client` service status
